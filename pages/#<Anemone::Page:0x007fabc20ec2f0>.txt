 ajouter un x-robots-tag avec nginx 
     administration serveur  
  
     blog seo  >  administration serveur  
 publié le 18 apr 2013 
 un article très court pour une fois. pour ceux qui passent de apache à nginx et qui souhaiteraient savoir comment ajouter un x-robots-tag dans le header http, voici quelle est la manière de procéder. 
 choisissez la(les) page(s) sur laquelle ajouter le champ, puis spécifiez les directives demandées. ce qui donne :     location   =   /ma_page   { 
  add_header   x-robots-tag   "noindex,   nofollow"  ;        
 } 
    bien entendu, vous pouvez définir des règles custom en fonction du user-agent. par exemple, si vous ne voulez envoyer un noindex qu'à googlebot, il faudra écrire dans votre fichier de configuration :     location   =   /ma_page   { 
  if   (  $http_user_agent   ~  *   googlebot)   { 
   add_header   x-robots-tag   "noindex,   nofollow"  ; 
  } 
 } 
    a noter que le ~* indique que vous souhaitez que le matching soit non sensible à la casse.   et voilà !        comments powered by  disqus
